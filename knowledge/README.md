# KNOWLEDGE BASE

Durable, reusable knowledge distilled from `research/` findings. While `research/` is chronological (one file per task), `knowledge/` is topical and long-lived — the org's institutional understanding.

## Rules
- One concept per file: `knowledge/<topic-slug>.md`.
- Link to the source finding(s) in `research/` and cross-link related notes (`[[topic]]`-style or relative links).
- No duplication: if it exists, update it; don't re-create.
- Keep claims tagged (FACT/LIKELY/SPECULATION/UNKNOWN) with sources.
- Supersede, don't silently overwrite — note what changed and why.

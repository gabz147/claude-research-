> **File contract** — Purpose: institutional memory of *why*. Owner: any role making a non-trivial choice. Update rules: append-only; never edit past entries (add a superseding entry instead). Recovery: read to avoid re-litigating settled choices. Format: dated ADR blocks. Lifecycle: permanent.

### ADR-001 — Repository is the source of truth (2026-05-31)
**Decision:** All durable state lives in markdown + JSON in this repo. Discord and any vector store are mirrors/observability only.
**Reasoning:** Claude sessions are ephemeral; conversation history is lost. Files survive resets, crashes, reboots.
**Alternatives:** Vector-DB-as-memory (claude-mem) — rejected as *authoritative* store (opaque, lossy, hard to audit). Discord-as-memory — rejected (not queryable/durable enough, rate-limited).
**Tradeoffs:** Files need disciplined updates each cycle. Accepted.
**Consequences:** Every cycle MUST update memory before exit. Bootstrap reads files, nothing else.

### ADR-002 — Headless full-auto via `claude -p --dangerously-skip-permissions` (2026-05-31)
**Decision:** Unattended cycles run Claude in print mode with permissions bypassed.
**Reasoning:** True 24/7 operation can't pause for human approval. User explicitly chose full-auto.
**Alternatives:** Scoped allowlist (stalls on unlisted actions); manual trigger (not autonomous).
**Tradeoffs:** Claude can run any tool. Mitigations: project-scoped CWD, lockfile, full logging, Discord alerts, git history for rollback.
**Consequences:** Supervisor must constrain CWD and log everything; human reviews via Discord + git.

### ADR-003 — systemd timer as scheduler (2026-05-31)
**Decision:** systemd service + timer drives the loop. Cron provided as fallback.
**Reasoning:** Survives reboot, restart-on-failure semantics, journald logging. `systemctl` present.
**Alternatives:** cron (no crash-restart); long-running daemon (more failure surface).
**Consequences:** Deployment installs a user/system unit; supervisor stays scheduler-agnostic.

### ADR-004 — Discord outbound via bot-token REST, not a gateway (2026-05-31)
**Decision:** Notifications are `curl` POSTs to the channels API with the bot token. No persistent gateway for observability.
**Reasoning:** Outbound-only needs no live socket; zero extra processes; trivially reliable across reboots.
**Alternatives:** Webhooks (need per-channel URLs; user supplied a token instead); full discord.py gateway (heavier — deferred to TASK-0006 for slash commands).
**Consequences:** A dedicated 🔬 RESEARCH ORG category with 6 channels was provisioned to isolate this project's output.

### ADR-006 — Mission set to Blue-Ocean opportunity discovery + stage-gated execution (2026-05-31)
**Decision:** The org's mission is to discover, validate, and execute a *small number of exceptional B2B business opportunities* (not generate idea lists). Adopted a 0–100 scoring rubric (Pain 20 / Spend 20 / Frequency 15 / AI-Leverage 15 / Competitive-Weakness 15 / Defensibility 15) with stage gates: <70 archive · 70–79 monitor · 80–89 validation project · 90+ execution project. Funnel: discover → validate → business model → MVP → build plan → customer acquisition.
**Reasoning:** Operator wants economic value, not research volume. Codifying the gate forces the org to *stop researching and start building* when confidence is high, avoiding perpetual-research failure mode.
**Alternatives:** Open-ended research (rejected — no destination); pure idea generation (rejected — explicitly a failure mode per MISSION).
**Tradeoffs:** Heavier doctrine + templates; narrower focus. Accepted — focus is the point.
**Consequences:** `prompts/opportunity-engine.md` is canonical doctrine; `runtime.md` advances the funnel (project-advance beats new discovery); `projects/` holds promoted opportunities; `research/OPPORTUNITIES.md` is the scored ledger; `templates/` holds stage artifacts; `#🏆-opportunities` Discord channel added. AI is a capability, never the product — score AI Leverage, don't chase AI.

### ADR-009 — First discovery cycle: OPP-001 MONITOR (not promoted); systemd timer left operator-gated (2026-06-15)
**Decision (a):** Scored the first opportunity (OPP-001, IRA prevailing-wage & apprenticeship compliance) at **79/100 → MONITOR**, deliberately *not* promoting to a project, despite very high Pain/Spend/Frequency.

TASK-0002 narrowed to that remaining step.
**Alternatives:** Promote OPP-001 anyway (rejected — over-fits the first idea, violates "try to kill it first"); auto-install the timer (rejected — unattended autonomy without explicit go-ahead).
**Consequences:** Ledger has its first row; no ≥80 project yet, so the funnel directs a 2nd discovery cycle next. Scoring used ADR-008 weights; TASK-0012 filed to fix the two artifacts (template + skill) that still show the old rubric.

### ADR-010 — Promote OPP-001b (IRA PWA audit-defense) to first VALIDATION project (2026-06-15)
**Decision:** Re-framed OPP-001 to its **policy-insulated audit-defense / penalty-remediation sub-wedge** (already-claimed 2023–2025 vintage credits), re-scored it **83/100 → VALIDATION**, and created `projects/01-ira-audit-defense/` — the org's **first promoted project**. Chose this over a 2nd net-new discovery cycle.
**Reasoning:** (a) Doctrine prefers advancing toward a buildable business over generating more ideas, and re-framing directly attacks the two caps that held OPP-001 at 79. (b) Fresh evidence *corroborated and strengthened* the load-bearing claim: OBBBA (signed 2025-07-04) is a **prospective** phase-out, **not a wholesale repeal** — already-claimed credits remain valid and auditable (≥4 independent Big-4/AmLaw/tax-policy sources); PWA persists (final regs 2024-06-25); **transferees are independently liable** for penalties, widening the buyer pool. The audit-defense market is served manually by Big-4/law firms with no productized product. (c) The re-score is honest, not a pump: Frequency was **downgraded** 13→10 (episodic per buyer) even as CompWk (8→11), Defens (10→12), AI (6→7), Pain (23→24) rose — net +4 to 83.
**Alternatives:** Run a 2nd discovery cycle in a new sector (rejected — leaves the strongest known opportunity un-advanced; the engine says narrow once something clears 80). Keep OPP-001 at MONITOR and only "watch" (rejected — the watch trigger, a policy-insulated re-frame, was met and verified this cycle).
**Tradeoffs / known weaknesses carried into validation:** MEDIUM confidence; promoted *to be disproven*, not to be built. Two open risks the VALIDATION_REPORT must resolve: (1) **demand latency** — real IRS PWA audit cadence is still single-vendor-sourced (Landgate/KPMG); (2) **finite runoff horizon** — the vintage base decays unless the engine generalizes to surviving regimes (45Q/45Z/45X, Davis-Bacon). Plus a **privilege** positioning question (compete vs sell-to-firms).
**Consequences:** Discovery narrows — next cycles drive project 01 down the funnel (VALIDATION_REPORT + market/customer/competitor analyses) rather than scanning new sectors, until validation either reaches HIGH (→ BUSINESS_MODEL) or kills it (→ archive, post-mortem). OPP-001 marked superseded-by-OPP-001b in the ledger. Build remains hard-gated at ≥90 + ADR-008 checklist.

### ADR-011 — Validation verdict for project 01-ira-audit-defense: HOLD (desk-research exhausted) (2026-06-16)
**Decision:** Ran VALIDATION MODE (TASK-0013) for the org's first project; produced VALIDATION_REPORT + MARKET/CUSTOMER/COMPETITOR analyses. Verdict = **HOLD / MORE EVIDENCE** (not PROCEED, not ARCHIVE). Did **not** advance to BUSINESS_MODEL; did **not** archive. Confidence stays MED.
**Reasoning:** The disprove pass **failed to kill** the bet and in two areas strengthened it — but it could not raise confidence to HIGH, and HIGH is the gate to BUSINESS_MODEL. (a) **Demand infrastructure is now FACT, not anecdote:** IRS released **Form 7220** (Rev. Dec 2025, standardized PWA attest/correct/penalty math, required per facility for TY2025) and **Form 4255** (PWA penalty/recapture reporting) — dedicated enforcement instruments, upgrading the prior single-vendor "penalty wave." (b) **Terminal value mitigated:** 45Q survives OBBBA (BOC <2033, PWA 5×), Davis-Bacon federal-contract audits are perennial (OMB Compliance Supplement sampling), FEOC adds a new clawback vector — the engine generalizes beyond the IRA runoff. (c) **But two load-bearing commercial unknowns are unresolvable by desk research:** willingness-to-pay for a *software-enabled* vendor vs privileged law firms (test 2), and actual IRS audit-*notice volume* (human half of test 1). (d) A **new commoditization risk** surfaced: Form 7220 self-correction + preventive incumbents creeping downstream (DSPTCH's PWA penalties calculator) narrow the defensible wedge to *contested post-notice audit defense + missing-record reconstruction*. (e) Honest counter-evidence: IRS enforcement funding cut **$45.6B → $24B** + TIGTA staffing cuts may suppress the human-audit rate (partly offset by self-attestation/document-matching).
**Alternatives:** PROCEED to BUSINESS_MODEL anyway (rejected — would build a model on an untested WTP assumption, violating mission's customer-evidence-over-analysis rule). ARCHIVE (rejected — nothing disproved the core bet; policy insulation + demand machinery confirmed). Keep grinding desk research (rejected — secondary sources are exhausted on the two open questions; further desk work has diminishing returns).
**Consequences:** Project 01 is HOLD, **operator-gated**: advancing requires primary discovery the autonomous org cannot do — 5–10 buyer/firm interviews (developer tax directors + Big-4/tax-controversy partners) to resolve WTP + compete-vs-channel, plus FOIA/TIGTA/§6418-dispute data for real audit cadence. Funnel implication: with the leading project blocked on operator input and no other ≥80 project, the next autonomous cycle should resume **discovery of a net-new opportunity** (runtime priority 3) rather than idle. TASK-0013 → completed; TASK-0014 (operator-gated primary discovery) filed.

### ADR-013 — Net-new discovery (vacuum-screened): OPP-003 (SEC cyber-materiality & disclosure-defense) MONITOR, not promoted (2026-06-16)
**Decision:** Ran a net-new discovery cycle (TASK-0016), this time **screening 4 candidate regimes for a productization vacuum BEFORE deep research** (applying the ADR-012 learning). Rejected EU CBAM (10+ mature emissions-data vendors) and EU AI Act conformity (crowded-forming AI-governance GRC: Credo AI/Holistic AI/OneTrust/IBM) at the screen. Advanced **SEC cyber-incident materiality-determination & disclosure-defense** because its problem-shape is a liability-laden *decision/defense* workflow (not a reporting form), which resists horizontal SaaS — the same shape as the org's only promoted win (OPP-001b). Scored **72/100 → MONITOR**, deliberately NOT promoted.
**Reasoning:** The disprove pass capped it honestly at three points. (1) **Policy/rescission risk (primary):** SEC Item 1.05 (4-day material-incident 8-K) is in effect as of June 2026 but faces active rescission petitions (ABA/Bank Policy Institute May 2025; NAM support) under a Republican-majority SEC (Chair Atkins) — the near-term forcing-function is at risk. Partial mitigant: the Oct-2024 enforcement vs Unisys/Avaya/Check Point/Mimecast ($7M total) rested on **anti-fraud + disclosure-controls (Section 13(a)/Reg S-K Item 106)**, not solely Item 1.05, so the materiality-record/consistency need survives a 1.05 rollback (OPP-001b-style policy insulation). (2) **No clean vacuum:** competition is *forming* — Kovrr (CRQ) and FAIR-MAM (adopted model) already occupy "materiality assessment"; only the narrower *legal-defensible contemporaneous record + disclosure-consistency engine* sub-wedge (GC/disclosure-committee buyer, distinct from the CISO-buyer CRQ tools) is open. Better than CBAM/OPP-002, not a true greenfield. (3) **Episodic frequency** (24 Item-1.05 filings in yr-1) → weak recurring-revenue logic unless repositioned as continuous disclosure-controls tooling. AI leverage is real and specific (consistency engine that catches the Unisys "hypothetical-risk-factor-vs-known-fact" failure; memo/8-K drafting; audit trail) but human-in-the-loop, 7/10.
**Alternatives:** Promote on raw Pain/Spend (rejected — over-fits; policy + competition + frequency all cap it). Deep-dive CBAM instead (rejected at the vacuum screen — crowded). Sharpen to the disclosure-controls sub-wedge and promote now à la OPP-001b (rejected — that re-frame needs the policy risk resolved + primary discovery that no CRQ vendor owns the legal-record wedge; logged as the watch-trigger instead).
**Consequences:** Ledger gains OPP-003 (MONITOR) with an explicit watch-trigger (Item 1.05 survives intact + securities-class-action wave makes the defense-record wedge acute, OR primary discovery confirms an unowned disclosure-controls sub-wedge). Still no second ≥80 project; project 01 remains operator-gated (TASK-0014). **Meta-learning (3 consecutive 70s — OPP-001 79, OPP-002 71, OPP-003 72 — vs OPP-001b 83):** once a compliance forcing-function is *public*, horizontal SaaS/consultancies close the vacuum within ~1–2 years; ≥80 requires either a regime whose forcing-function *just* landed (vacuum still open) or a sharp *defensive sub-wedge* incumbents structurally can't serve (the OPP-001b pattern). Filed as a LOW continuous-improvement task to bias future discovery toward those two shapes.
